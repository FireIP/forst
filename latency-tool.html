🔁 Approach 1 — Loopback (pixel marker)
  1. The Source Canvas draws animated content plus a large black/white square in the top-left corner that toggles state every N ms (configurable).
  2. Exactly when the state changes, t_src = performance.now() is recorded.
  3. The canvas stream is fed into a local RTCPeerConnection pair (pc1 → pc2), passing through the chosen codec's encoder and decoder entirely in the browser.
  4. A requestAnimationFrame loop reads the remote <video> into a tiny <canvas> each frame, computes average luminance of the marker region, and detects the transition.
  5. When detected, t_play = performance.now() is recorded and latency = t_play − t_src is added to the sample set.
  6. Statistics (mean, median, p95, p99, stddev) and a histogram are updated live.

This measures: encoder + RTP packetise + decoder + rAF render latency — the full browser WebRTC pipeline without network.

📡 Approach 2 — External stream (getStats)
  1. Enter your OME WebRTC URL (e.g. wss://host:3334/app/stream). OvenPlayer connects and plays the stream.
  2. The tool monkey-patches RTCPeerConnection to capture the underlying PC and calls pc.getStats() every 500 ms.
  3. Key stats displayed: jitter buffer delay, avg decode time per frame, FPS, packet loss, NACK count, network RTT, and estimated playout latency.
  4. Optional pixel detection: configure the marker region to match where OBS injects a toggling B/W overlay in the scene. The tool logs each transition time.
  5. Optional WebSocket sync: run the included helper script on the same machine. It controls OBS via obs-websocket and sends {"state":N,"tSrc":ms} events so absolute latency can be computed as in Approach 1.
🧮 Statistics explained
MeanAverage latency over all samples.
Median50th percentile — robust to occasional spikes.
p95 / p9995th / 99th percentile — worst-case tails.
StdDevJitter: how much latency varies between samples.
Jitter buffer delay From getStats: jitterBufferDelay / jitterBufferEmittedCount × 1000. The average time frames spend in the jitter buffer before playback.
Est. playout latency Date.now() − estimatedPlayoutTimestamp. Rough estimate (requires sender/receiver clocks to be NTP-synced).
🛠 OBS marker setup (for Approach 2)

To inject a toggling timing marker into your OBS scene:

  1. Add a Color Source (white, 200×200 px) in the top-left corner of your scene.
  2. Use an OBS Script (Python or Lua) to toggle the source's visibility every N ms and emit {"state":N,"tSrc":ms} over a local WebSocket server.
  3. Configure the marker region on the External Stream tab to match where the white square appears in the decoded video.
  4. Connect the WS sync URL on the External Stream tab so this page receives t_src events.

Without the script, pixel detection still logs transition times — useful for qualitative comparison or for benchmarks where you control both sides.
